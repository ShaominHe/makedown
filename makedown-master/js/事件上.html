事件上

事件：window中有一个对象，专门用来纪律事件的详细信息，包括鼠标坐标，键盘键码。。。
事件三要素：事件源+事件名称+事件处理程序
事件名称：触发了什么事件（点击click事件）
事件处理程序：事件触发后要执行的代码（函数形式）
事件的三种写法
1、window.onkeydown=function(e){
console.log(e)
}
2、在IE低版本中不兼容
btn.onclick=function(){
console.log(window.event)
}
3、兼容所有浏览器得到写法
btn.onclick=function(e){
var ev=e || window.event
console.log(ev)
}

事件对象

一、点击事件的鼠标坐标点
1、从元素内部开始计算的坐标：’offsetX’和’offsetY’
2、相对于浏览器的坐标：‘clientX’和‘clientY’
3、不管页面滚动到哪里都是根据窗口来计算坐标，不会随着滚动条的滚动而发生改变
相对于页面的坐标，会随着滚动条的滚动而加大，使用：’pageX’和’pageY’

二、鼠标点击按键信息
获取按键信息使用：”ev.button”
如果是0表示左键，如果是2表示右键
鼠标右键事件：”contextmenu”

事件类型

三、事件类型
在事件对象中有个一个属性type可以获取到当前事件的类型。
window.onload = function(e){
var ev = e || window.event;
console.log(ev.type); // load
}

六、键盘事件

七、表单事件
submit 提交表单
focus 获取焦点
blur 失去焦点
change 内容改变并失去焦点
input input的value值改变
onpropertychange 低版本IE中的input事件
补充：change和input都可用来监听文本框内容发生改变，但是change需要失去焦点的时候才能触发，而input是即使触发，input在低版本中不兼容用onpropertychange替代

绑定事件

八、事件的绑定方式
使用on来绑定事件有弊端；同一个事件只能给一个元素绑定一次，下面的会把上面的覆盖
使用addEventListener(type,handle,false)
参数1：给元素绑定的事件类型：如：click，mouseover…
参数2：处理事件的函数
参数3：是否在冒泡阶段执行，true在捕获阶段执行，false在冒泡阶段执行

例：
《button id=”btn”>按钮《/button>
《/body>
《script>
document.getElementById(“btn”).addEventListener(“click”,handle,false);
function handle(){
console.log(“第一次点击”);
}
document.getElementById(“btn”).addEventListener(“click”,handle1,false);
function handle1(){
console.log(“第二次点击”);
}

结果是：第一次点击
第二次点击

在IE低版本的浏览器中不兼容，在IE低版本中使用attachEvent（）代替
btn.attachEvent(“onclick”,one)
btn.attachEvent(“onclick”,two)
