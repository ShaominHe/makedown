事件下

事件流：事件发生的时候都会有一个触发并执行的过程
事件流分为三个阶段：捕获阶段，目标阶段，冒泡阶段
捕获阶段：事件开始由顶层元素触发，然后逐级向下传播，依次执行
目标阶段：触发当前自身的事件
冒泡阶段：事件由目标元素先接受，然后逐级向上传播，达到顶层元素，依次执行
事件默认在冒泡阶段执行
注意：
如果某个元素事件在捕获阶段被执行，则不会再冒泡阶段执行，即事件只会在事件流中执行一次

oDiv.addEventListener(“click”,handle,ture)
最后一个参数：是否在捕获阶段执行
ture 在捕获阶段执行
false 为默认的冒泡阶段执行

阻止冒泡事件
e.stopPropagetion();阻止事件冒泡
IE中：e.cancelBubble=true

默认行为
如a标签的跳转，form表单的提交，鼠标右键的菜单

1、阻止a标签:
第一种：
《a href=”javascript:;”》《/a》
第二种：
return false;
第三种：
document.querySelector(“form”).onclick=function(){
var e=e||window.event
e.preventDefault()
}

2、阻止form
第一种：《form action=’javascript:;’》
第二种：document.querySelector(“form”).onsubmit=function(){
var e=e||window.event
e.preventDefault()
}
第三种：将e.preventDefault换位return false

3、阻止右键：
第一种：
return false
第二种：
e.preventDefault()

IE中阻止默认事件：
e.returnvalue =false

事件委托
将元素的事件交给父元素绑定，通过父元素识别点击的目标元素处理对应的事件
e.target找精准的事件目标
IE中要用srcElement代替

通用绑定事件的方法
bind 给元素绑定事件
ele 将要绑定事件的元素
type 将要绑定的事件的类型
handle 处理该事件的函数
如果使用的是attachEvent,那么按顺序绑定的事件会倒叙执行

function bind(ele,type,handle){
try{
ele.addEventListener(type,handle,false)
}catch(e){
ele.attachEvent(“on”+type,handle)
}
}

封装的通用事件的解绑
function unbind(ele,type,handle){
try{
ele.removeEventListener(type,handle,false)
}catch(e){
ele.detachEvent(“on”+type,handle)
}
}
