变为工厂函数

age=age;
}
return obj
}
var obj1=createObj(“张三”,12)
var obj2=createObj(“李三”,12)

工厂函数可以批量对象
缺点：不能分辨具体的对象类型（对象前面没有名字）
解决方法使用构造函数

二、构造函数——-Math Date MouseclickEvent MouseoverEvent
function fn(name,age){
this.name=name;
this.age=age;
}
var obj=new fn(“张三”,12)

使用构造函数创建对象的时候发生了什么？
1、函数中创建了一个空函数
2、将函数中的this指向这个空对象
3、执行函数中代码（为这个空对象添加属性）
4、返回这个对象

构造函数的要求：
1、构造函数必须和new配合使用，否则就没有创建对象的功能
2、构造函数中不允许有return，如果在构造函数中return了一个基本类型，new的时候不会生效，如果return了一个复杂类型，构造函数失去意义
3、通常情况构造函数的首字母大写
4、如果构造函数不需要参数，那么new的时候可以省略小括号

三、原型
任何函数都有在定义的时候，系统都会给他分配一个空间，使用天生自带的prototype的属性访问到这个空间，这个空间存的是一个对象
函数可以当做对象用
function fn(){}
console.log(fn.prototype) //访问到的空间叫做原型

function Person(name,age){
this.name=name;
this.age=age;
}
Person.prototype.sex=’女’
var obj = new Person(“张三”,20)
console.log(obj,sex) // 女
通过new 创建出来的这个对象默认可以访问到构造函数的原型对象上的属性和方法
当对象访问属性的时候，先从自己找，找不到就去原型上去找
function Person(name,age){
this.name=name;
this.age=age;
}
Person.prototype.eeaatt=function(){
console.log(this.name+”能吃”)
}
funcrion eeaatt(){
console.log(this.name+”能吃”)
}
var obj1=new.Person(“张三”,20)
obj1.eeaatt(); // 张三能吃
eeaatt() //能吃
var obj2=new.Person(“李三”,20)
console/log(obj1.eeaatt==obj2.eeaatt) //true

创建对象：自定义构造函数
需要属性：通过构造函数得到参数传入
需要方法：通过构造函数的原型绑定

四、访问器
所有的原型对象都有一个constructor属性，可以访问到构造函数
function Person(name,age){
this.name=name;
this.age=age;
}
访问Person这个构造函数的原型
console.log(Person.prototype)
访问Person的原型的constructor
console.log(Person.prototype.constructor)
对比一下 Person的原型的constructor和Person
console.log(Person.prototype.constructor==Person) //true

五、原型链
任何对象天生都自带一个属性__proto__，他可以访问构造函数对应的原型
function Person(name,age){
this.name=name;
this.age=age;
}
var obj= new Person(“张三”,20)
访问obj的构造函数对应的原型
console.log(obj.proto)
访问构造函数的原型
console.log(Person.prototype)
对比 构造函数的原型和对象的proto
console.log(Person.prototype ==obj.proto) //true

var Personyuanxing = obj.proto //Person的原型是一个对象，这个对象肯定也有也有proto

访问原型的原型对应的构造函数
console.log(Personyuanxing.proto.constructor) //Object

Object对应的原型
var objectyuanxing = Personyuanxing.proto;

Object对应的原型的原型
console.log(objectyuanxing.proto) //null


这就是原型链
作用：对象访问属性，首先在自己内部找，自己内部找不到，就找原型，原型上也没有，就在原型的原型上找，在原型链上找，找到null也没有找到，就返回undefined。
对象设置属性和原型无关只有访问有关
